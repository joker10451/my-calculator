# Кросс-браузерное тестирование

## Обзор

Этот документ описывает процесс кросс-браузерного тестирования блога на разных браузерах.

## Поддерживаемые браузеры

### Desktop
- ✅ Chrome (Chromium)
- ✅ Firefox
- ✅ Safari (WebKit)
- ✅ Edge

### Mobile
- ✅ Chrome Mobile (Android)
- ✅ Safari Mobile (iOS)
- ✅ Tablet (iPad Pro)

## Команды для запуска тестов

### Все браузеры
```bash
npm run test:e2e
```

### Отдельные браузеры
```bash
# Chrome
npm run test:e2e:chrome

# Firefox
npm run test:e2e:firefox

# Safari
npm run test:e2e:safari

# Edge
npm run test:e2e:edge

# Мобильные браузеры
npm run test:e2e:mobile
```

### UI режим (интерактивный)
```bash
npm run test:e2e:ui
```

### Просмотр отчета
```bash
npm run test:e2e:report
```

## Известные проблемы и решения

### Chrome
- **Проблема**: Нет известных проблем
- **Решение**: N/A

### Firefox
- **Проблема**: Возможны небольшие различия в рендеринге CSS
- **Решение**: Проверить визуально, использовать кросс-браузерные CSS свойства

### Safari (WebKit)
- **Проблема**: Некоторые современные CSS функции могут не поддерживаться
- **Решение**: Использовать fallback значения, проверить на реальном устройстве

### Edge
- **Проблема**: Требуется установленный браузер Edge
- **Решение**: Установить Edge или пропустить тесты для Edge

## Чеклист тестирования

### Функциональность
- [ ] Поиск статей работает
- [ ] Фильтрация по категориям работает
- [ ] Навигация между статьями работает
- [ ] Прогресс чтения отображается
- [ ] Оглавление (TOC) работает
- [ ] Шеринг в соцсети работает
- [ ] Комментарии отправляются
- [ ] Клики на калькуляторы работают

### Визуальное тестирование
- [ ] Шрифты загружаются корректно
- [ ] Изображения отображаются
- [ ] Цвета соответствуют дизайну
- [ ] Анимации работают плавно
- [ ] Нет визуальных артефактов

### Производительность
- [ ] Страница загружается < 3 секунд
- [ ] Нет блокирующих запросов
- [ ] Изображения оптимизированы
- [ ] JavaScript не блокирует рендеринг

## Отчеты о тестировании

После запуска тестов отчеты сохраняются в:
- `test-results/` - JSON отчеты
- `playwright-report/` - HTML отчет

Откройте HTML отчет командой:
```bash
npm run test:e2e:report
```

## Автоматизация в CI/CD

Тесты автоматически запускаются в CI при:
- Push в main ветку
- Pull Request
- Релизе новой версии

Конфигурация CI находится в `.github/workflows/`
