# Требования к интеграции Яндекс Кешбэка

## Введение

Интеграция партнерской ссылки Яндекс Кешбэка в сайт калькуляторов для предоставления пользователям дополнительной возможности экономии. Интеграция должна быть деликатной, полезной и не отталкивающей пользователей.

## Глоссарь

- **Система**: Веб-приложение калькуляторов Считай.RU
- **Кешбэк_Блок**: Компонент для отображения информации о кешбэке
- **Партнерская_Ссылка**: https://ya.cc/8WF6y8
- **Контекстный_Совет**: Рекомендация по использованию кешбэка в контексте конкретного калькулятора

## Требования

### Требование 1: Информационный блок в футере

**Пользовательская история:** Как пользователь, я хочу видеть полезные сервисы в футере сайта, чтобы узнать о дополнительных возможностях экономии.

#### Критерии приемки

1. КОГДА пользователь прокручивает страницу до футера, ТО Система ДОЛЖНА отображать секцию "Полезные сервисы"
2. В секции "Полезные сервисы" Система ДОЛЖНА показывать информацию о Яндекс Кешбэке с иконкой и кратким описанием
3. КОГДА пользователь нажимает на блок кешбэка, ТО Система ДОЛЖНА открыть Партнерскую_Ссылку в новой вкладке
4. Кешбэк_Блок ДОЛЖЕН иметь стиль, соответствующий общему дизайну сайта
5. Текст блока ДОЛЖЕН быть информативным и не рекламным: "Экономьте на покупках с Яндекс Кешбэком"

### Требование 2: Контекстные советы в калькуляторах

**Пользовательская история:** Как пользователь финансовых калькуляторов, я хочу получать полезные советы по экономии, связанные с моими расчетами.

#### Критерии приемки

1. КОГДА пользователь получает результат в кредитном калькуляторе, ТО Система ДОЛЖНА показывать Контекстный_Совет о возможности использования кешбэка для досрочного погашения
2. КОГДА пользователь получает результат в ипотечном калькуляторе, ТО Система ДОЛЖНА показывать Контекстный_Совет о накоплении средств через кешбэк
3. КОГДА пользователь получает результат в зарплатном калькуляторе, ТО Система ДОЛЖНА показывать Контекстный_Совет о дополнительном доходе через кешбэк
4. Контекстный_Совет ДОЛЖЕН содержать ссылку на Партнерскую_Ссылку
5. Контекстный_Совет ДОЛЖЕН быть оформлен как информационная подсказка, а не как реклама

### Требование 3: Отслеживание эффективности

**Пользовательская история:** Как владелец сайта, я хочу понимать эффективность размещения партнерских ссылок, чтобы оптимизировать их расположение.

#### Критерии приемки

1. КОГДА пользователь нажимает на любую ссылку кешбэка, ТО Система ДОЛЖНА отправить событие в Яндекс.Метрику
2. События ДОЛЖНЫ различаться по источнику: "cashback_footer", "cashback_credit", "cashback_mortgage", "cashback_salary"
3. КОГДА происходит клик по ссылке кешбэка, ТО Система ДОЛЖНА добавить UTM-метки для отслеживания источника трафика

### Требование 4: Адаптивность и производительность

**Пользовательская история:** Как пользователь мобильного устройства, я хочу, чтобы информация о кешбэке корректно отображалась на всех устройствах.

#### Критерии приемки

1. Кешбэк_Блок ДОЛЖЕН корректно отображаться на мобильных устройствах
2. Контекстные_Советы ДОЛЖНЫ адаптироваться под размер экрана
3. КОГДА загружается страница с кешбэк элементами, ТО время загрузки НЕ ДОЛЖНО увеличиваться более чем на 50мс
4. Все кешбэк элементы ДОЛЖНЫ быть доступны для пользователей с ограниченными возможностями (ARIA-метки)

### Требование 5: Соответствие рекламному законодательству

**Пользовательская история:** Как владелец сайта, я хочу соблюдать требования законодательства о рекламе, чтобы избежать правовых проблем.

#### Критерии приемки

1. Все блоки с партнерскими ссылками ДОЛЖНЫ содержать пометку "Реклама" или "Партнерская ссылка"
2. КОГДА отображается информация о кешбэке, ТО она ДОЛЖНА быть честной и не содержать ложных обещаний
3. Партнерские ссылки ДОЛЖНЫ иметь атрибут rel="sponsored nofollow"
4. Информация о кешбэке НЕ ДОЛЖНА имитировать основной контент сайта
