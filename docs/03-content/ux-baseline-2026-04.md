# Baseline UX и конверсий (апрель 2026)

Цель baseline: зафиксировать стартовые метрики до серии UX-итераций и сравнивать динамику по неделям.

## Что снимаем еженедельно

- `/blog/`: scroll depth 25/50/75/90, поиск и фильтры, пустые результаты.
- `/offers/`: фильтрация, поиск, пустые результаты, клики CTA.
- Калькуляторы: раскрытие FAQ, просмотры trust-блока, клики в офферы из result block.

## Источники данных

- Yandex Metrika (цели `ux_*` и `referral_*`)
- GA4 (события `ux_*` и `referral_*`)
- Внутреннее хранилище (`localStorage`): ключ `ux_events`

## Техническая фиксация baseline

- В проект добавлен единый трекер UX-событий: `src/lib/analytics/uxMetrics.ts`.
- На странице каталога офферов выводится оперативный baseline-снимок по локальным UX-событиям.
- Для A/B CTA используется закрепление варианта в `localStorage` (`ab_cta_<product>`).

## Минимальные KPI на квартал

- Рост доли сессий со scroll depth >= 50% на ключевых страницах.
- Рост CTR партнёрских CTA из блоков после калькуляторов.
- Снижение доли пустых результатов в блоге и каталоге офферов.
