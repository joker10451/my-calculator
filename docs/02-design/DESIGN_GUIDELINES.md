# DESIGN GUIDELINES

Короткий регламент визуального стиля для сайта `schitay-online.ru`.

## 1) Визуальная модель

- Стиль: `dark premium` (референсы: Apple/Spotify/Netflix).
- Иерархия: контент и действия важнее декора.
- Принцип: 1 основной акцент (`primary`) + 1 нейтральный вторичный CTA.

## 2) Базовые токены (использовать в первую очередь)

- Поверхности:
  - `surface-card`
  - `surface-card-hover`
  - `surface-muted`
- Секции:
  - `section-shell`
  - `section-title`
  - `section-subtitle`
- Кнопки:
  - `cta-primary`
  - `cta-secondary`
- Поля:
  - `field-control`
  - `field-control-lg`
- Бейджи:
  - `chip-muted`
  - `chip-accent`

## 3) Компонентные правила

- Карточки: всегда через `surface-card`/`surface-muted`, без случайных градиентов.
- Формы: инпуты и селекты только с `field-control`.
- CTA: не более 1 primary-кнопки в блоке первого экрана.
- Текст:
  - заголовки: `font-black` для главных секций;
  - вторичный текст: `text-slate-300`/`text-slate-400` в dark.

## 4) Do / Don’t

- Do:
  - Использовать единые радиусы (xl/2xl/3xl).
  - Держать одинаковые отступы между секциями.
  - Проверять контраст текста на темном фоне.
- Don’t:
  - Не добавлять яркие разноцветные градиенты без UX-причины.
  - Не смешивать старые utility-классы со стилем токенов в одном блоке.
  - Не делать несколько конкурирующих primary CTA рядом.

## 5) Чеклист перед merge

- Проверка desktop/tablet/mobile для:
  - `Header`, `Hero`, каталогов, карточек и форм.
- Нет «светлых пятен» и инородных блоков на dark-фоне.
- `npm run lint` без ошибок.
- `npm run build` проходит (или зафиксирован внешний сбой окружения).
