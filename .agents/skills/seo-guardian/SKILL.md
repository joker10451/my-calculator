# Навык: SEO Guardian (Считай.RU)

Этот навык предназначен для обеспечения максимальной видимости сайта в поисковых системах через автоматизацию генерации статики и проверку метаданных.

## Когда использовать

- После любого изменения контента в калькуляторах или блоге.
- После изменения структуры заголовков, мета-тегов или изображений.
- Каждую неделю для профилактического аудита.

## Рабочий процесс

### 1. Подготовка окружения
Перед проверкой убедитесь, что проект собран:
```bash
npm run build
```

### 2. Генерация статических страниц
Запустите скрипт создания HTML-файлов для каждой страницы. Это КРИТИЧЕСКИ ВАЖНО для индексации SPA:
```bash
node scripts/generate-static-html.js
```

### 3. Обновление ресурсов индексации
Обновите карту сайта и проверьте robots.txt:
```bash
node scripts/generate-sitemap.js
```

### 4. Комплексная проверка (Валидация)
Запустите ваш кастомный SEO-чекер:
```bash
node scripts/validateSEOCompliance.cjs
```

## Чек-лист проверки (Guardrails)

- [ ] **Статус 200**: Убедитесь, что все сгенерированные HTML-файлы открываются без ошибок.
- [ ] **Favicons**: Все иконки должны присутствовать и быть прописаны в HTML (ICO перед SVG).
- [ ] **Meta Tags**: Проверьте, что Title и Description на сгенерированных страницах соответствуют списку в `generate-static-html.js`.
- [ ] **No 2025**: Убедитесь, что в мета-тегах нет упоминаний прошлого года.

## Авто-действия при ошибках

- Если `validateSEOCompliance.cjs` показывает **❌ Ошибки**, немедленно исправьте их перед деплоем.
- Если отсутствуют Apple Touch Icons, используйте `scripts/generateAppleTouchIcon.js`.
