# Навык: Calculator Tester (Считай.RU)

Этот навык предназначен для обеспечения 100% точности финансовых и математических расчетов во всех калькуляторах.

## Когда использовать

- Перед финальной сборкой (`build`).
- Сразу после изменения формул в `src/lib/`.
- При добавлении новых полей в формы калькуляторов.

## Рабочий процесс

### 1. Модульное тестирование (Unit Tests)
Запустите тесты логики:
```bash
npm run test
```
*Если тесты падают — исправление имеет наивысший приоритет.*

### 2. E2E тестирование (UI)
Проверьте, что интерактивные элементы (кнопки, ползунки) работают корректно:
```bash
npm run test:e2e
```

### 3. Проверка покрытия (Coverage)
Убедитесь, что критические модули (налоги, кредиты, госпошлины) покрыты тестами на **90%** и выше:
```bash
npx vitest run --coverage
```

### 4. Стресс-тестирование вводов
Используйте `fast-check` (если настроен) или вручную проверьте:
- Максимально возможные суммы.
- Нулевые значения.
- Некорректные символы (буквы в полях для цифр).
- Огромные числа (переполнение).

## Guardrails (Ограничения)

- [ ] **Не игнорировать `@ts-ignore`**: В тестах это допускается только в крайних случаях.
- [ ] **Регрессия**: Новый тест должен быть добавлен для каждого найденного и исправленного бага.
- [ ] **Сверку с законом**: Все критические финансовые тесты должны иметь ссылку на закон или официальный калькулятор (ЦБ РФ, ФНС) в комментариях.
